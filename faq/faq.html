1 如何使用Github和jekyll搭建blog

1.1 申请github帐号

在Github官网上申请一个github帐号，并新建一个repository，名称为gyq（可以自己定义）。

1.2 安装Git客户端 (Linux)

$sudo apt-get install git git-gui

1.3 安装jekyll

$sudo apt-get install jekyll

1.4 生成密钥，项目可以push到github上的先决条件

$ssh-keygen -t rsa -C "xxx@163.com"

1.5 将生成的key添加到github中去。

Account Setting -> SSH keys -> ADD SSH key

用编辑器打开idrsa.pub，将内容复制到key的输入框里.

1.6 测试与github通信是否正常

$ssh git@github.com

如果配置正确，显示

ERROR: Hi xxx! You've successfully authenticated, but GitHub does not provide shell access Connection to github.com closed.

1.7 设置用户

$git config –global user.name "xxx"

$git config –global user.email xxx@gmail.com

1.8 创建本地项目目录

$mkdir gyq

$cd gyq

$git init

$touch README

$git add README

$git commit -m 'first commit'

$git remote add origin git@github.com:xxx/gyq.git

$git push origin master

1.9 更新文件

$git commit -a

$git push origin master

1.10 创建和合并分支

$git branch 显示当前分支

$git branch new-branch 创建分支

$git checkout new-branch 切换到新分支

$git merge new-branch 合并到master

$git branch

$git push

1.11 日常发布命令

$git clone git@github.com:xxx/xxx.github.com.git//本地如果无远程代码，先做这步，不然就忽略

$cd .ssh/xxx.github.com//定位到你blog的目录下

$git pull origin master //先同步远程文件，后面的参数会自动连接你远程的文件

$git status //查看本地自己修改了多少文件

$git add .//添加远程不存在的git文件

$git commit * -m "what I want told to someone"

$git push origin master //更新到远程服务器上
